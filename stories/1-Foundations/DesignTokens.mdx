# Design Tokens

Design tokens are the visual design atoms of the design system. They define the core visual language: colors, typography, spacing, and more.

## CSS Architecture

The Elevate Design System uses a modular CSS architecture for maintainability:

```
src/css/
├── main.css                # Entry point + themes (sqli-light, sqli-dark)
├── _fonts.css              # TWK Everett font declarations
├── _tokens.css             # Design tokens (brand colors, typography)
├── _tokens-standalone.css  # Standalone tokens (HEX, no Tailwind dependency)
├── _customizations.css     # Component overrides (focus, links, toggles)
└── _utilities.css          # Custom utilities (sr-only, reduced motion)
```

  background-color: var(--color-sqli-cream);

<div class="bg-sqli-cream text-sqli-midnight">

</div>

## Why OKLCH?

We use **OKLCH** color format instead of HEX in theme definitions:

### Benefits

1. **Perceptually uniform** - Colors look consistent across the spectrum
2. **Accessibility** - Easier to calculate contrast ratios
3. **Modern standard** - Supported by all modern browsers
4. **Better manipulation** - Easy to create tints/shades programmatically

### OKLCH Format

```
oklch(Lightness% Chroma Hue)
```

## Custom Utilities

### Screen Reader Only

Hide content visually while keeping it accessible to screen readers:

```html
<button>
  <svg>...</svg>
  <span class="sqli-sr-only">Close menu</span>
</button>
```

### Reduced Motion

The design system automatically respects `prefers-reduced-motion`:

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

## Governance

### Adding New Tokens

1. Define the token in `_tokens.css` (or `main.css` for theme-specific tokens)
2. Document it in this page
3. Update `SQLI-BRAND-REFERENCE.md` if it's a brand color
4. Run `npm run validate:colors` to verify consistency

### Modifying Colors

When changing brand colors:

1. Update the HEX value in `SQLI-BRAND-REFERENCE.md`
2. Convert to OKLCH using [oklch.com](https://oklch.com)
3. Update `_tokens.css` (HEX) and `main.css` (OKLCH theme values)
4. Run validation to ensure consistency
