## Where to Place Contributions

### Decision Tree

1. **Is it a design token or brand asset?** -> **Foundations**
2. **Is it a new React component?** -> `src/components/<ComponentName>/`
3. **Is it a new CSS component story?** -> `stories/2-Components/<Category>/`
4. **Is it a custom CSS override?** -> `src/css/main.css`

### Component Categories

See the [Component Index](?path=/docs/governance-component-index--docs) for the complete list of 25 React components + CSS utility components organized by category (Actions, Data Display, Data Input, Feedback, Navigation, Layout, Mockup).

## Naming Conventions

### React Component Pattern

Components use CVA variants with the `cn()` utility:

```tsx
import { forwardRef } from 'react';
import { cva, type VariantProps } from 'class-variance-authority';
import { cn } from '../../lib/utils';

const myVariants = cva('base-classes', {
  variants: {
    variant: { primary: '...', secondary: '...' },
    size: { sm: '...', md: '...', lg: '...' },
  },
  defaultVariants: { variant: 'primary', size: 'md' },
});

export interface MyProps
  extends React.HTMLAttributes<HTMLElement>,
    VariantProps<typeof myVariants> {}

export const MyComponent = forwardRef<HTMLElement, MyProps>(
  ({ className, variant, size, ...props }, ref) => (
    <div ref={ref} className={cn(myVariants({ variant, size }), className)} {...props} />
  )
);
MyComponent.displayName = 'MyComponent';
```

### File Naming

- Component files: `src/components/<ComponentName>/<ComponentName>.tsx` + `index.ts`
- Story files: `stories/2-Components/<Category>/ComponentName.stories.tsx`
- CSS overrides: Add to `src/css/main.css`

## Theme System

The design system uses `data-theme` attribute for theming (NOT `.dark` class). The `ElevateTheme` React provider manages themes programmatically.

Available themes: `sqli-light` (default) and `sqli-dark`.

### Custom CSS Overrides

Add overrides in `src/css/main.css` using CSS variables:

```css
/* Light mode override */
.component:focus {
  border-color: var(--color-focus) !important;
}

/* Dark mode variant */
[data-theme='sqli-dark'] .component:focus {
  border-color: var(--color-focus) !important;
}
```

## Story Template

```tsx
import React from 'react';
import type { Meta, StoryObj } from '@storybook/react';
import { MyComponent } from '../../../src/components/MyComponent';

const meta: Meta<typeof MyComponent> = {
  title: 'Components/<Category>/MyComponent',
  component: MyComponent,
  tags: ['autodocs'],
  parameters: {
    docs: {
      description: {
        component: `
React **MyComponent** with SQLI themes.

## Usage
\`\`\`tsx
import { MyComponent } from '@sqli/elevate-design-system';

<MyComponent variant="primary">Content</MyComponent>
\`\`\`
        `,
      },
    },
  },
};

export default meta;
type Story = StoryObj<typeof MyComponent>;

export const Default: Story = {
  args: { children: 'Default', variant: 'primary' },
};

export const Variants = () => (
  <div className="flex gap-4">
    <MyComponent variant="primary">Primary</MyComponent>
    <MyComponent variant="secondary">Secondary</MyComponent>
  </div>
);

export const DarkMode = () => (
  <div data-theme="sqli-dark" className="bg-[var(--color-base-100)] p-6 rounded-lg">
    <MyComponent variant="primary">Dark Mode</MyComponent>
  </div>
);
DarkMode.parameters = { backgrounds: { default: 'dark' } };
```

## Accessibility Checklist

All components must meet WCAG 2.1 AA standards. See [Introduction - Accessibility](?path=/docs/introduction--docs#accessibility) for design system accessibility features.

Before submitting a component:

- [ ] Color contrast meets WCAG 2.1 AA (4.5:1 for text, 3:1 for UI)
- [ ] Keyboard navigation supported (Tab, Enter, Space, Escape)
- [ ] ARIA attributes properly set where needed
- [ ] Focus states visible and clear
- [ ] Screen reader tested (or semantically correct HTML used)
- [ ] Reduced motion respected (`prefers-reduced-motion`)

## Responsive Expectations

Components should be tested at all breakpoints (Mobile 375px, Tablet 768px, Desktop 1280px, Desktop XL 1920px). See [Introduction - Responsive Design](?path=/docs/introduction--docs#responsive-design) for the complete breakpoint reference.

## Documentation Requirements

Every component story must include:

1. **Description**: What the component is and when to use it
2. **Variants**: All available color variants documented
3. **Sizes**: All size options shown
4. **States**: All interactive states demonstrated
5. **Dark Mode**: Rendering in dark theme
6. **Usage**: Code examples with React component imports and props

## Review Process

1. **Design Review**: UX team validates design consistency
2. **Code Review**: DS maintainers review implementation
3. **Accessibility Audit**: a11y compliance verified
4. **Documentation Check**: All required docs present
5. **Visual Regression**: No unintended visual changes

### PR Description Template

Your PR description should include:

1. **Summary**: What does this PR do?
2. **Changes**: List of changes made
3. **Testing**: How was this tested?
4. **Screenshots**: Visual changes (if applicable)
5. **Related Issues**: Link to related issues

### Before Merge Checklist

- [ ] Code follows the style guidelines
- [ ] Self-review completed
- [ ] Documentation updated
- [ ] Tests pass locally
- [ ] No console errors
- [ ] Storybook builds successfully
- [ ] Accessibility verified
