# Framework Guide -- Angular

Integrate the Elevate Design System into Angular projects. This guide covers Angular CLI setup with standalone components and signals.

---

## Prerequisites

- Node.js >= 22.0.0
- npm >= 9.0.0
- Angular CLI >= 17

```bash
npm install git+https://github.com/sqli/elevate-design-system.git
```

---

## Setup with Angular CLI

### 1. Create the project

```bash
ng new my-app --style=css --routing
cd my-app
npm install git+https://github.com/sqli/elevate-design-system.git
```

### 2. Configure angular.json

Add the design system CSS to the `styles` array in your `angular.json`. This ensures the CSS is bundled and loaded globally:

```json
{
  "projects": {
    "my-app": {
      "architect": {
        "build": {
          "options": {
            "styles": [
              "node_modules/@sqli/elevate-design-system/dist/fonts.css",
              "node_modules/@sqli/elevate-design-system/dist/sqli-design-system.css",
              "src/styles.css"
            ]
          }
        }
      }
    }
  }
}
```

### 3. Import CSS in styles.css (alternative)

Instead of adding to `angular.json`, you can import directly in your global stylesheet:

```css
/* src/styles.css */
@import '@sqli/elevate-design-system';
@import '@sqli/elevate-design-system/fonts';

body {
  @apply bg-base-100 text-base-content;
}
```

Both approaches work. The `angular.json` method gives you explicit control over load order. The CSS import method is simpler and consistent with other frameworks.

### 4. Configure the theme

Set the `data-theme` attribute on the `<html>` element in `src/index.html`:

```html
<!doctype html>
<html lang="en" data-theme="sqli-light">
  <head>
    <meta charset="utf-8" />
    <title>My App</title>
    <base href="/" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
  </head>
  <body>
    <app-root></app-root>
  </body>
</html>
```

### 5. Verify the setup

Update your `AppComponent` to confirm the design system is loaded:

```ts
// src/app/app.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  standalone: true,
  template: `
    <div class="min-h-screen bg-base-100 text-base-content p-8">
      <h1 class="text-3xl font-bold mb-4">Elevate Design System</h1>
      <button class="btn btn-primary">It works</button>
    </div>
  `,
})
export class AppComponent {}
```

## Wrapper Components

The design system is CSS-only. You apply DaisyUI class names directly in your templates. The following wrapper components provide a typed API using `@Input` decorators for the most common patterns.

### ButtonComponent

```ts
// src/app/components/button/button.component.ts
import { Component, Input } from '@angular/core';
import { NgClass } from '@angular/common';

type ButtonVariant = 'primary' | 'secondary' | 'accent' | 'ghost' | 'link' | 'outline';
type ButtonSize = 'lg' | 'md' | 'sm' | 'xs';

@Component({
  selector: 'app-button',
  standalone: true,
  imports: [NgClass],
  template: `
    <button
      class="btn"
      [ngClass]="[variantClass, sizeClass, loading ? 'loading' : '', wide ? 'btn-wide' : '']"
      [disabled]="disabled || loading"
    >
      <ng-content />
    </button>
  `,
})
export class ButtonComponent {
  @Input() variant: ButtonVariant = 'primary';
  @Input() size: ButtonSize = 'md';
  @Input() loading = false;
  @Input() wide = false;
  @Input() disabled = false;

  private readonly variantMap: Record<ButtonVariant, string> = {
    primary: 'btn-primary',
    secondary: 'btn-secondary',
    accent: 'btn-accent',
    ghost: 'btn-ghost',
    link: 'btn-link',
    outline: 'btn-outline',
  };

  private readonly sizeMap: Record<ButtonSize, string> = {
    lg: 'btn-lg',
    md: 'btn-md',
    sm: 'btn-sm',
    xs: 'btn-xs',
  };

  get variantClass(): string {
    return this.variantMap[this.variant];
  }

  get sizeClass(): string {
    return this.sizeMap[this.size];
  }
}
```

**Usage:**

```html
<app-button variant="primary" size="lg">Save Changes</app-button>
<app-button variant="ghost" size="sm">Cancel</app-button>
<app-button variant="outline" [loading]="true">Submitting</app-button>
```

## Theme Switching with Angular Service

### ThemeService

```ts
// src/app/services/theme.service.ts
import { Injectable, signal, computed, effect } from '@angular/core';

export type Theme = 'sqli-light' | 'sqli-dark';

const STORAGE_KEY = 'elevate-theme';

@Injectable({ providedIn: 'root' })
export class ThemeService {
  private readonly themeSignal = signal<Theme>(this.getInitialTheme());

  readonly theme = this.themeSignal.asReadonly();
  readonly isDark = computed(() => this.themeSignal() === 'sqli-dark');

  constructor() {
    effect(() => {
      const current = this.themeSignal();
      document.documentElement.setAttribute('data-theme', current);
      localStorage.setItem(STORAGE_KEY, current);
    });

    if (typeof window !== 'undefined') {
      const mediaQuery = window.matchMedia('(prefers-color-scheme: dark)');
      mediaQuery.addEventListener('change', (e) => {
        const stored = localStorage.getItem(STORAGE_KEY);
        if (!stored) {
          this.themeSignal.set(e.matches ? 'sqli-dark' : 'sqli-light');
        }
      });
    }
  }

  toggleTheme(): void {
    this.themeSignal.update((current) =>
      current === 'sqli-light' ? 'sqli-dark' : 'sqli-light'
    );
  }

  setTheme(theme: Theme): void {
    this.themeSignal.set(theme);
  }

  private getInitialTheme(): Theme {
    if (typeof window === 'undefined') return 'sqli-light';

    const stored = localStorage.getItem(STORAGE_KEY);
    if (stored === 'sqli-light' || stored === 'sqli-dark') return stored;

    const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
    return prefersDark ? 'sqli-dark' : 'sqli-light';
  }
}
```

## Using the Tailwind Preset

If your Angular project uses Tailwind CSS 4, you can extend it with SQLI brand tokens.

### 1. Install Tailwind CSS

```bash
npm install -D tailwindcss@^4.1.0
```

### 2. Configure Tailwind

```javascript
// tailwind.config.js
import sqliPreset from '@sqli/elevate-design-system/preset';

export default {
  presets: [sqliPreset],
  content: ['./src/**/*.{html,ts}'],
};
```

### 3. Use preset utilities in templates

This adds the following utilities to your Tailwind classes:

| Utility | Example Class | Value |
|---------|---------------|-------|
| Colors | `bg-sqli-cobalt`, `text-sqli-cream` | SQLI brand colors |
| Font Family | `font-sans`, `font-everett` | TWK Everett |
| Border Radius | `rounded-sqli-sm`, `rounded-sqli-lg` | Brand-consistent rounding |
| Shadows | `shadow-sqli-md` | Brand shadows |
| Animations | `animate-sqli-fade-in` | Brand transitions |

```html
<!-- Use preset utilities alongside DaisyUI classes -->
<div class="bg-sqli-cream text-sqli-midnight font-everett">
  <h1 class="text-sqli-cobalt">Branded Heading</h1>
  <div class="rounded-sqli-lg shadow-sqli-md p-6">
    <button class="btn btn-primary">DaisyUI Button</button>
  </div>
</div>
```

## ViewEncapsulation Considerations

Angular components use `ViewEncapsulation.Emulated` by default, which scopes styles to each component. Since the Elevate Design System uses global CSS classes, there are two approaches to ensure DaisyUI classes work correctly:

### Approach 1: Global styles (recommended)

Import the design system CSS globally via `angular.json` or `styles.css` as shown in the setup section. DaisyUI classes applied directly in component templates will work because Angular does not encapsulate classes applied to elements in the template -- only styles defined in the component's `styles` or `styleUrls` are scoped.

### Approach 2: Disable encapsulation per component

If you encounter styling issues in deeply nested components, you can disable encapsulation:

```ts
import { Component, ViewEncapsulation } from '@angular/core';

@Component({
  selector: 'app-my-component',
  standalone: true,
  encapsulation: ViewEncapsulation.None,
  template: `
    <button class="btn btn-primary">Styled Correctly</button>
  `,
})
export class MyComponent {}
```

Use this sparingly. In most cases, global CSS import is sufficient.

## Server-Side Rendering (Angular Universal)

When using Angular SSR, the theme must be applied carefully to avoid a flash of unstyled content:

### 1. Set a default theme in index.html

```html
<html lang="en" data-theme="sqli-light">
```

### 2. Apply theme before Angular hydration

Add a blocking script in `index.html` to apply the stored theme before the page renders:

```html
<head>
  <script>
    (function() {
      var theme = localStorage.getItem('elevate-theme');
      if (theme === 'sqli-light' || theme === 'sqli-dark') {
        document.documentElement.setAttribute('data-theme', theme);
      }
    })();
  </script>
</head>
```

This runs synchronously before Angular hydrates, preventing a theme flash.
