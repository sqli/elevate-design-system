- **Storybook Chromatic** (planned): Automated visual regression testing
- **Snapshot testing**: For all component variants
- **Cross-browser visual comparison**: Ensuring consistency across browsers

## Testing Checklist by Component Type

### Primitives

- [ ] All variants render correctly
- [ ] All states (hover, focus, disabled, etc.) work
- [ ] Responsive behavior at all breakpoints
- [ ] Dark mode appearance
- [ ] Accessibility audit passes
- [ ] No visual regressions

### Compositions

- [ ] All primitive dependencies work correctly
- [ ] Layout adapts to content variations
- [ ] Responsive behavior
- [ ] Interactive states function properly
- [ ] Accessibility audit passes

## Release Checklist

Before any release:

- [ ] All stories render without errors
- [ ] Visual regression tests pass (or changes approved)
- [ ] Accessibility audit complete (0 critical issues)
- [ ] Cross-browser testing complete
- [ ] Documentation updated
- [ ] Changelog entry added
- [ ] Version bumped appropriately
- [ ] Build succeeds without warnings

## Reporting Issues

When reporting a bug:

1. **Component name**: Which component is affected
2. **Browser/OS**: Where you observed the issue
3. **Steps to reproduce**: How to trigger the bug
4. **Expected vs actual**: What should happen vs what happens
5. **Screenshots**: Visual evidence if applicable
